Stacks



支持动态使用数据堆栈来处理硬件中断。要使用该环境变量，请将其放在 Config.nt 文件中。


语法



stacks=n,s 


参数



	n

	指定堆栈数。n 的有效值为 0 以及 8 到 64 之间的数字。

	s

	指定每个堆栈的大小（按字节计）。s 的有效值为 0 以及 32 到 512 之间的数字。




注释


 默认设置
	
 下表显示了 stack 命令的默认设置。


计算机 
Stacks 


IBM PC、IBM PC/XT、IBM 便携式 PC 
0,0 


其他
9,128 


    
 堆栈分配的特殊情况
	
 接收硬件中断后，Windows XP 从指定数量的堆栈中分配一个堆栈。指定 n 和 s 值为 0 时，Windows XP 将不分配堆栈。如果值为 0，每个正在运行的程序必须有足够的堆栈空间来容纳计算机的硬件中断驱动程序。n 和 s 的值为 0时，许多计算机都可以正常运行，保留一些内存供程序使用。但如果将这些值设置为 0 后，计算机变得不稳定，请返回到默认值。
    
    



范例



 要为每个硬件中断处理分配 8 个 512 字节的堆栈，请将以下命令添加到 Config.nt 文件中： 

 stacks=8,512
